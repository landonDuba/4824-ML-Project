Phase 1 - Exploratory Data Analysis (Complete)

What I did:
Phase 1 was the exploratory phase. The goal was to understand the data and prove the
idea works before writing any real algorithms.

Step 1 - Built the data pipeline
- The dataset has a dedicated All-Star Selections.csv with clean labels
- Created a lag-join: take a player's stats from season Y-1, label = were they an All-Star in Y
- This is the core framing: predict next year's All-Stars from last year's stats
- Handled traded players by keeping only the season-total (TOT) row

Step 2 - Measured class imbalance
- Only ~24 out of ~450 players per season are All-Stars = 1:11 imbalance
- A model predicting "never All-Star" would be 94% accurate but completely useless
- This means accuracy is a bad metric, need F1 and top-24 recall instead

Step 3 - Ranked features by correlation
- VORP (0.58) and Win Shares (0.54) are the strongest predictors
- PTS/game (0.53) and MP/game (0.41) also matter
- Shooting efficiency (FG%, FT%) barely correlates, volume and impact matter more

Step 4 - Ran a throwaway baseline
- Untuned Logistic Regression to confirm the signal is there
- ROC-AUC: 0.97 (strong signal in the data)
- Top-24 recall per season: only 50-72% (real room to improve)
- Precision: 0.40 (too many false positives which is the main problem to fix in Phase 2)

What this proved:
- The data pipeline works and the lag-join framing is valid
- The data has strong predictive signal
- A properly tuned model should do significantly better than the baseline
